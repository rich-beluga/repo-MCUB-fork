# OpenAgent 0.4.2 Changelog

## Исправления

- Исправлена нестабильная работа inline-кнопок `Отмена`, `Очистить` и `Регенерировать`.
- Исправлена зависимость кнопок OpenAgent от общего inline/Hikka callback слоя.
- Исправлена потеря callback payload при генерации кнопок.
- Исправлены устаревающие callback-записи: добавлена собственная таблица callback payload для OpenAgent.
- Исправлена обработка нажатий на кнопки через прямой `events.CallbackQuery(pattern=b"^oa:")` handler.
- Исправлена очистка контекста чата через inline-кнопку `Очистить`.
- Исправлена регенерация ответа с сохранением исходного `prompt`, `full_prompt` и вложений.
- Исправлена отмена генерации через кнопку `Отмена` без зависимости от внешних callback wrappers.
- Исправлена обработка устаревших inline-кнопок: теперь пользователь получает понятное уведомление.
- Исправлены проблемы с профилями: profile tool теперь получает расширенную информацию через `GetFullUserRequest`.
- Исправлено неполное отображение данных профиля пользователя.
- Исправлено отсутствие локального аватара профиля: аватар скачивается в `openagent_profiles/profile_<id>.jpg`, если доступен.
- Исправлено отсутствие информации об общих чатах: profile tool добавляет список общих чатов, если Telegram API возвращает данные.
- Исправлены ответы на вопросы вида «кто это» при reply: агент получает профиль автора reply-сообщения и использует его как приоритетный контекст.
- Исправлена обработка длинных ответов: слишком длинный ответ отправляется файлом `openagent_answer.txt`.
- Исправлены слишком длинные live-status сообщения: значения tool status и `Agent Log` обрезаются до безопасной длины.
- Исправлено отображение HTML-заголовка ответа: теги `<i>` и `<b>` больше не показываются как обычный текст.
- Исправлена очистка служебных tool-injection строк из финального ответа.
- Исправлена защита от вывода внутренних инструкций вида `Use the above message...` и `call the task tool...`.
- Исправлена динамическая рабочая директория terminal tool: больше нет hardcoded `/home/codespace/MCUB-fork`.
- Исправлено создание директории skills: используется `./openagent_skills` от текущей рабочей директории запуска MCUB.
- Исправлена поддержка reply media: текстовые файлы читаются как текст, изображения передаются в vision payload.
- Исправлена поддержка `.webm`/video stickers: первый кадр извлекается через `ffmpeg` и отправляется модели как изображение.
- Исправлена обработка неподдерживаемых или больших media: агент получает метаданные вместо падения.
- Исправлены retry-сценарии для OpenAI-compatible API, когда модель не поддерживает `temperature`.
- Исправлены retry-сценарии для OpenAI-compatible API, когда модель требует `max_completion_tokens` вместо `max_tokens` или наоборот.
- Исправлена стабильность `web_search` для URL: агент может читать HTML-страницы и получать очищенный текст/ссылки.
- Исправлена работа с контекстом чата: история хранится отдельно по `chat_id` и ограничивается `context_turns`.
- Исправлена регенерация с учётом текущих вложений и исходного контекста.
- Исправлена отправка итогового ответа отдельным сообщением вместо редактирования `Thinking...`.
- Исправлено удаление `Thinking...` после успешного ответа.
- Исправлена совместимость команд `.skills`, `.imss`, `.sendss`, `.delss` с локальным storage skills.
- Исправлена отправка `.md` skill-файлов через `.sendss`.
- Исправлена загрузка skills в system prompt: используются только доступные `.md` файлы из `openagent_skills`.
- Исправлена стабильность MCUB helper mode `.mcubh`: используется отдельный system prompt учителя MCUB.
- Исправлены ошибки при отсутствии API key: сообщение стало понятнее и указывает на `.cfg OpenAgent api_key`.
- Исправлены fallback-сценарии Telegram tools: большинство tools возвращают текстовую ошибку вместо падения модуля.

## Технические детали

- Версия модуля обновлена до `0.4.2`.
- Модуль проверен через `python -m py_compile`.
- Файл `OpenAgent-MCUB-repo.py` сохранён с repo metadata для установки из `repo-MCUB-fork`.
